Doctor Appointment Booking System – Complete Task Description 
(Backend + Frontend)
 You are required to build a full-stack application that includes backend development (.Net) and 
frontend UI (REACT). The system will allow patients to book doctor appointments, doctors to 
manage slots, and admins to monitor all bookings.
 Modules to Implement
 1. Doctor Registration
 Develop both backend and frontend components that allow doctors to sign up with:
 • Name
 • Email
 • Password
 • Specialization
 The data must be saved in the system database, and a simple UI form should be provided for the 
signup.
 2. Doctor Slot Management
 Create backend APIs and corresponding frontend screens where doctors can:
 • Add available appointment slots
 • Date
 • Start Time
 • End Time
 All slot data should be stored in the database and displayed in the frontend.
 3. Patient Registration
 Provide backend APIs and frontend UI for patient signup with:
 • Name
 • Email
 • Password
 • Phone Number
 Both backend logic and frontend form validations must be implemented.
4. Appointment Booking
 Patients should be able to:
 • Choose a doctor
 • Select a visible list of available slots
 Once booked:
 • Slot should be marked as Booked in the database
 • The frontend UI must reflect the updated status
 5. Admin Appointment Overview
 Create an admin dashboard (frontend) supported by backend APIs to display:
 • Appointment ID
 • Patient Name
 • Doctor Name
 • Appointment Date & Time
 • Status (Booked / Visited)
 6. Doctor Updates Appointment Status
 Doctors should have a UI screen where they can update the appointment status:
 • Booked → Visited
 Backend API must update the status, and UI should reflect it.
 Final Requirement
 You must develop:
 • Backend (APIs + DB models)
 • Frontend (UI screens for all modules)
 Bonus Points
 Pagination, search and filtering for tables.
